2. Все файлы проекта — подробный разбор

2.1 calc.py — главный сервер

#!/usr/bin/env python3
import http.server, json, os, threading, time
from prometheus_client import start_http_server, Counter, Gauge, Info

# Метрики Prometheus:
requests_total = Counter('app_requests_total', 'Total requests', ['method', 'path', 'status'])
active_requests = Gauge('app_active_requests', 'Active requests')
app_info = Info('app_info', 'Application info')
app_info.info({'version': os.getenv('APP_VERSION', 'dev'), 'build': os.getenv('BUILD_URL', '')})

DRUID_HOST = os.getenv('DRUID_HOST', 'druid-broker.infra.svc.cluster.local')
DRUID_PORT = os.getenv('DRUID_PORT', '8082')

class Handler(http.server.BaseHTTPRequestHandler):
    def do_GET(self):
        active_requests.inc()
        if self.path == '/metrics':
            self.send_response(200)
            self.send_header('Content-Type', 'text/plain; version=0.0.4')
            self.end_headers()
            self.wfile.write(generate_latest().encode())
        else:
            self.send_response(200)
            self.send_header('Content-Type', 'text/html')
            self.end_headers()
            html = f'''<h1>Сбер-Мониторинг v {os.getenv('APP_VERSION', '🔥')}</h1>
<p>Druid: {DRUID_HOST}:{DRUID_PORT}</p>
<p>Build: {os.getenv('BUILD_URL', '')}</p>'''
            self.wfile.write(html.encode())
            requests_total.labels(method='GET', path=self.path, status=200).inc()
        active_requests.dec()

# Запуск: HTTP на 8080, метрики на 8000
start_http_server(8000)
server = http.server.HTTPServer(('0.0.0.0', 8080), Handler)
server.serve_forever()

Что делает:

  • Слушает порт 8080 — возвращает HTML страницу
  • Слушает порт 8000 — возвращает Prometheus метрики в формате text/plain
  • Собирает метрики: requests_total (счётчик), active_requests (датчик), app_info (инфо)
  • Читает переменные окружения: DRUID_HOST, DRUID_PORT, APP_VERSION, BUILD_URL

Что делает:

  • Копирует calc.py и test_app.py в /tmp/

  • Т.к. в OpenShift образ собирается через BuildConfig, файлы копируются из репозитория

2.5 Jenkinsfile — CI/CD пайплайн

pipeline {
    agent any
    options {
        buildDiscarder(logRotator(numToKeepStr: "10"))  // хранить 10 последних сборок
        timeout(time: 1, unit: "HOURS")                  // таймаут пайплайна 1 час
        timestamps()                                      // показывать время в логах
    }
    environment {
        OPENSHIFT_API = 'https://api.rm3.7wse.p1.openshiftapps.com:6443'
        OS_TOKEN = credentials('openshift-token')    // ID в Jenkins Credentials
        NAMESPACE = 'art2010901-dev'
        APP_NAME = 'sber-monitoring'
        APP_VERSION = "${BUILD_NUMBER}"              // номер сборки Jenkins
        IMAGE = "image-registry.openshift-image-registry.svc:5000/${NAMESPACE}/${APP_NAME}:${BUILD_NUMBER}"
    }
    stages {
        stage('Checkout') {
            steps { checkout scm }                   // клонировать Git репозиторий
        }
        stage('Test') {
            steps {
                sh 'python3 -m venv venv'
                sh '. venv/bin/activate && pip install -r requirements.txt'
                sh '. venv/bin/activate && python test_app.py'
            }
        }
        stage('Login OpenShift') {
            steps {
                sh """
                    oc login --token=${OS_TOKEN} --server=${OPENSHIFT_API} --insecure-skip-tls-verify=true
                    oc project ${NAMESPACE}
                """
            }
        }
        stage('Apply K8s Core') {
            steps {
                sh """
                    oc apply -f k8s/deployment.yaml -n ${NAMESPACE}
                    oc apply -f k8s/service.yaml -n ${NAMESPACE}
                    oc apply -f k8s/route.yaml -n ${NAMESPACE}
                """
            }
        }
        stage('Apply Monitoring') {
            steps {
                sh """
                    oc apply -f k8s/servicemonitor.yaml -n ${NAMESPACE}
                    oc apply -f k8s/prometheusrule.yaml -n ${NAMESPACE}
                    oc apply -f k8s/prometheus.yaml -n ${NAMESPACE}
                    oc apply -f k8s/grafana/configmap.yaml -n ${NAMESPACE}
                    oc apply -f k8s/grafana/deployment.yaml -n ${NAMESPACE}
                    oc apply -f k8s/grafana/service.yaml -n ${NAMESPACE}
                    oc apply -f k8s/grafana/route.yaml -n ${NAMESPACE}
                """
            }
        }
        stage('Build Image') {
            steps {
                script {
                    sh """
                        oc new-build --name=${APP_NAME} --binary --strategy=docker -n ${NAMESPACE} || true
                        oc start-build ${APP_NAME} --from-dir=. --wait -n ${NAMESPACE}
                        oc tag ${NAMESPACE}/${APP_NAME}:latest ${NAMESPACE}/${APP_NAME}:${BUILD_NUMBER} -n ${NAMESPACE}
                    """
                }
            }
        }
        stage('Deploy to TEST') {
            steps {
                sh """
                    oc set image deployment/${APP_NAME}-test app=${IMAGE} -n ${NAMESPACE}
                    oc set env deployment/${APP_NAME}-test APP_VERSION=${APP_VERSION} BUILD_URL=${BUILD_URL} -n ${NAMESPACE}
                    oc scale deployment/${APP_NAME}-test --replicas=1 -n ${NAMESPACE}
                """
            }
        }
        stage('Approve PROD') {
            options { timeout(time: 1, unit: "DAYS") }
            steps {
                input message: "Deploy build #${BUILD_NUMBER} to PROD?", ok: "Yes, deploy!"
            }
        }
        stage('Deploy to PROD') {
            steps {
                sh """
                    oc set image deployment/${APP_NAME}-prod app=${IMAGE} -n ${NAMESPACE}
                    oc set env deployment/${APP_NAME}-prod APP_VERSION=${APP_VERSION} BUILD_URL=${BUILD_URL} -n ${NAMESPACE}
                    oc scale deployment/${APP_NAME}-prod --replicas=2 -n ${NAMESPACE}
                """
            }
        }
    }
    post {
        success { echo "=== SUCCESS: Build #${BUILD_NUMBER} deployed ===" }
        failure { echo "=== FAILED: Build #${BUILD_NUMBER} ===" }
    }
}

Ключевые моменты:

  • replicas: 0 — деплой создан, но поды не запущены. Jenkins сам решает когда их включать
  • command — при запуске контейнера устанавливается prometheus_client и скачивается calc.py с GitHub. Это временное решение, пока Jenkins не соберёт образ
  • containerPort: 8080 (http) — порт для веб-трафика
  • containerPort: 8000 (metrics) — порт для Prometheus метрик
  • DRUID_HOST/DRUID_PORT — переменные окружения для подключения к Druid (аналитическая БД)
  • APP_VERSION/BUILD_URL — заполняются Jenkins при деплое

3.2 k8s/service.yaml

apiVersion: v1
kind: Service
metadata:
  name: sber-monitoring
  namespace: art2010901-dev
  labels:
    app: sber-monitoring-prod
spec:
  selector:
    app: sber-monitoring-prod       # ← ВАЖНО: отправлять трафик к подам с этим label
  ports:
  - name: http
    port: 8080                      # Порт сервиса
    targetPort: 8080                # Порт контейнера (containerPort в deployment)
  - name: metrics
    port: 8000
    targetPort: 8000
  type: ClusterIP                   # Внутренний IP, не доступен из интернета

Как работает Service:

  • Создаёт стабильный IP-адрес (ClusterIP) внутри кластера
  • По selector app: sber-monitoring-prod находит все поды с таким label
  • Балансирует трафик между подами (round-robin)
  • Порт 8080 → трафик от Route или других сервисов
  • Порт 8000 → Prometheus скрейпит метрики

3.3 k8s/route.yaml

apiVersion: route.openshift.io/v1
kind: Route
metadata:
  name: sber-monitoring-route
  namespace: art2010901-dev
spec:
  to:
    kind: Service
    name: sber-monitoring            # → направляет трафик в Service
  port:
    targetPort: http                 # → на порт http (8080)
  tls:
    termination: edge                # TLS заканчивается на роутере (HTTPS → HTTP)
  wildcardPolicy: None

Как работает Route:

  • OpenShift Router (HAProxy) слушает внешние запросы
  • Route создаёт DNS-имя: sber-monitoring-route-art2010901-dev.apps.rm3.7wse.p1.openshiftapps.com
  • edge termination: HTTPS трафик расшифровывается на роутере, до пода идёт HTTP
  • Трафик: Браузер → HTTPS → Route → Service:8080 → Pod:8080

Почему standalone, а не Prometheus Operator CR?

  • В DevSandbox нет прав на ClusterRole/ClusterRoleBinding — они нужны для Prometheus Operator
  • Prometheus CR был создан, но оператор не среагировал (нет прав, нет StatefulSet)
  • Standalone Prometheus через Deployment + ConfigMap работает без дополнительных прав
  • ConfigMap монтируется как файл /etc/prometheus/prometheus.yml
  • Prometheus скрейпит sber-monitoring:8000 — это metrics порт Service
  • Rules загружаются из /etc/prometheus/rules/alerts.yml

Типы TLS termination:

  • edge — HTTPS заканчивается на роутере, до пода HTTP (мы используем этот)

8. Типовые проблемы и их решение

ПроблемаСимптомПричинаРешение
Pod CrashLoopBackOff Pod перезапускается, статус CrashLoopBackOff Команда в контейнере завершается с ошибкой oc logs <pod> --tail=30 — смотреть последние строки ошибки.
Чаще всего: pip не может установить prometheus_client, или не может скачать calc.py (нет curl).
Решение: проверить command в deployment, установить curl или использовать python urllib.
ImagePullBackOff Pod не стартует, статус ImagePullBackOff Образ не найден в registry oc describe pod <pod> — смотреть событие Failed to pull image.
Решение: запустить oc start-build sber-monitoring --from-dir=. --wait чтобы собрать образ.
Site not accessible Route открывается, но Connection Refused или 503 Поды не Running, или Service selector не совпадает oc get pods — проверить Running ли поды.
oc describe service sber-monitoring — проверить Endpoints (должны быть IP подов).
Если Endpoints пустые — selector не совпадает с labels подов.
Jenkins 403 Jenkins показывает 403 Forbidden OAuth проблема или CSRF Открыть в браузере, войти через "Log in with OpenShift".
Если не помогает — перезапустить Jenkins: oc rollout restart deploymentconfig jenkins.
Jenkins pipeline fails on Test Этап Test падает с ошибкой test_app.py не может найти calc.py Проверить test_app.py: пути к файлам. Запустить локально python test_app.py.
Если нет python, установить: winget install --id Python.Python.3.9.
Build fails oc start-build падает с ошибкой Dockerfile не найден или ошибка сборки oc logs build/sber-monitoring-1 — смотреть ошибку.
Проверить что Dockerfile есть в корне репозитория.
Prometheus targets DOWN Prometheus UI показывает target DOWN Service sber-monitoring:8000 не отвечает oc get svc sber-monitoring — провери что порт 8000 есть.
oc exec <pod> -- curl localhost:8000/metrics — проверить что под отдаёт метрики.
Grafana "No data" Дашборды пустые Datasource URL неверный Grafana → Configuration → Data Sources → Prometheus → URL: http://prometheus:9090.
Нажать "Save & Test" — должно быть зелёное "Data source is working".
403 on ServiceMonitor Cannot create ServiceMonitor Нет прав на мониторинг ресурсы Используем standalone Prometheus (k8s/prometheus.yaml), не Prometheus Operator.
ServiceMonitor и PrometheusRule не нужны — удалите их из Jenkinsfile если падает.
oc login expired oc команды возвращают "forbidden" Токен истёк (DevSandbox токены живут ~24ч) Получить новый токен в OpenShift Console → ваш профиль → Copy login command.
Обновить в Jenkins Credentials.
Pipeline stuck at Approve Pipeline висит на этапе Approve PROD Ждёт ручного подтверждения Открыть Jenkins → открыть сборку → Нажать "Yes, deploy!"
